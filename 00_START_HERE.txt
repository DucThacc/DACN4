⚠️ CRITICAL: IP CONFIGURATION POINTS
═════════════════════════════════════════════════════════════════════════

4 PLACES to update before deploying on Ubuntu Server:

1. web-waf/.env
   Change: DVWA_HOST=localhost
   To:     DVWA_HOST=192.168.1.100 (Your Ubuntu IP)

2. suricata/.env
   Change: NETWORK_INTERFACE=eth0
   To:     NETWORK_INTERFACE=<your interface> (eth0, ens0, etc)

3. suricata/docker-compose.yml
   Change: command: suricata ... -i eth0
   To:     command: suricata ... -i <your interface>

4. suricata/configs/suricata.yaml
   Change: interface: eth0 (appears twice)
   To:     interface: <your interface>

📖 See IP_CONFIGURATION.md for detailed instructions!


🚀 QUICK START (3 STEPS)
═════════════════════════════════════════════════════════════════════════

Step 1: On Ubuntu Server, get your configuration
  $ ubuntu_ip=$(hostname -I | awk '{print $1}')
  $ interface=$(ip route | grep default | awk '{print $5}')
  $ echo "IP: $ubuntu_ip, Interface: $interface"

Step 2: Update all IP addresses (see IP_CONFIGURATION.md)
  OR use automated script: ./configure.sh

Step 3: Deploy!
  $ bash deploy.sh

That's it! 🎉

🎓 DEPLOYMENT SCENARIOS
═════════════════════════════════════════════════════════════════════════

Scenario A: First-time deployment (IDS mode - detection only)
  1. Update IPs in 4 locations
  2. Run: bash deploy.sh
  3. Access DVWA at http://192.168.1.100/
  4. Run tests: bash web-waf/scripts/test_sqli.sh
  5. View logs: tail -f suricata/logs/eve.json

Scenario B: Want IPS mode (blocking attacks)
  1. Complete Scenario A first
  2. Keep system running
  3. Run: sudo bash deploy.sh ips
  4. Answer prompts (setup iptables rules)
  5. Test blocking: attacks should be dropped

Scenario C: Just exploring (local machine, not Ubuntu)
  1. Keep IPs as localhost/127.0.0.1
  2. Run: bash deploy.sh
  3. Access DVWA at http://localhost/
  4. System works but can't test network attacks

Scenario D: In production (after testing)
  1. Review all logs for false positives
  2. Tune paranoia levels in ModSecurity
  3. Update/filter Suricata rules as needed
  4. Set up log aggregation/SIEM if desired
  5. Monitor 24/7
