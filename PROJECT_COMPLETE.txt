                    🎯 THE 4 IP CONFIGURATION POINTS
═════════════════════════════════════════════════════════════════════════

Before deploying on Ubuntu Server, update EXACTLY 4 places:

1️⃣  web-waf/.env
    Line: DVWA_HOST=localhost
    Change to: DVWA_HOST=192.168.1.100  (your Ubuntu Server IP)

2️⃣  suricata/.env
    Line: NETWORK_INTERFACE=eth0
    Change to: NETWORK_INTERFACE=<your_interface>

3️⃣  suricata/docker-compose.yml
    Line 35 approx: -i eth0
    Change to: -i <your_interface>

4️⃣  suricata/configs/suricata.yaml
    Lines 30, 95, 98: eth0 and 172.20.0.0/16
    Change interface to: <your_interface>
    Change HOME_NET if needed

📖 See IP_CONFIGURATION.md for detailed step-by-step instructions!


═════════════════════════════════════════════════════════════════════════
                         🚀 DEPLOYMENT IN 3 STEPS
═════════════════════════════════════════════════════════════════════════

Step 1: Prepare Ubuntu Server
  $ ubuntu_ip=$(hostname -I | awk '{print $1}')
  $ interface=$(ip route | grep default | awk '{print $5}')
  $ echo "IP: $ubuntu_ip, Interface: $interface"

Step 2: Update configuration (4 places - see above)
  $ nano web-waf/.env
  $ nano suricata/.env
  $ nano suricata/docker-compose.yml
  $ nano suricata/configs/suricata.yaml

Step 3: Deploy!
  $ bash deploy.sh

That's it! 🎉
